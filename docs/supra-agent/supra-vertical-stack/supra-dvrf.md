# Supra dVRF — Decentralized Verifiable Random Function

## What It Is
Supra dVRF provides tamper-proof, on-chain verifiable randomness as a native protocol primitive. Unlike third-party VRF services, it's embedded directly into Supra's L1 — no external dependencies.

## Architecture
- Uses a **threshold cryptographic architecture** — fully decentralized, not reliant on any single party.
- Features **output-privacy**, which enhances utility in blockchain applications and significantly reduces gas costs.
- Randomness is generated by a decentralized committee and verified on-chain — no single node can predict or manipulate the outcome.

## Use Cases
- **GameFi**: Loot drops, randomized encounters, card draws, battle outcomes
- **Lotteries & raffles**: Provably fair winner selection
- **NFT minting**: Random trait assignment, fair distribution
- **DeFi**: Random selection for governance, validator rotation
- **Any application** requiring provably fair, unpredictable outcomes

## Why It Matters for Your dApp
If your dApp needs randomness of any kind, using Supra dVRF means:
- No trust assumptions on an external oracle for randomness
- On-chain verifiability — anyone can prove the randomness wasn't tampered with
- Lower gas costs thanks to output-privacy design
- Native integration — no extra bridge calls or latency

## Integration
dVRF is accessed through the Supra Move framework. Your smart contract requests randomness, and the dVRF committee generates and delivers a verifiable random value to your contract.

See Supra developer docs for integration guides: https://docs.supra.com
